## 候选反例列表

每一条都以"如果 X 场景下 LoB fail，那 DTA 需要 Y 修正"的形式写。

### C1 · 超长上下文 prefill

**场景**：128k / 1M token prompt。
**假设风险**：prompt 本体就得跨边界传，128k tokens × 4 B/token embedding ≈ 500 KB，其实很小。但真实场景是**首次 embedding 推理会不会把外部链路占满**？

### C2 · 实时视频多模态

**场景**：4K@30fps 实时 vision-language 推理，raw 帧送进 Track A。
**假设风险**：3840×2160×3B × 30fps ≈ 750 MB/s，USB4 (5 GB/s) 够，但会挤占其它 I/O；USB 3.2 (0.625 GB/s) 直接爆。
**修正方向**：Host 侧放一个小型 pre-encoder（VAE / CLIP visual），把每帧压到 ~KB 级 embedding 再送。
**结论**：**LoB 需要"每模态一个小型 host-side pre-encoder"作为补丁。**这个补丁是廉价的（几百 MB VRAM 就够），可接受。

### C3 · 紧耦合 agentic 流水线

**场景**：AI 决策 → host 执行工具 → 结果回到 AI → 再决策。每一步都跨边界。
**假设风险**：边界穿越次数 = O(工具调用次数)，不是 O(1)。如果每步都要传大 tensor 或长历史，链路延迟就是瓶颈。
**修正方向**：状态最小化——只传结构化 JSON / 短文本，不传激活。
**结论**：TODO 量化。

### C4 · 模型热切换

**场景**：用户从 70B 模型切到 32B 模型，权重要重新装载。
**假设风险**：70B q4 ≈ 35 GB，USB4 5 GB/s → 7 秒装载。USB 3.2 0.625 GB/s → 56 秒。
**修正方向**：（a）Track A 内部装大容量 flash 存多个模型，热切换在模块内完成；（b）接受首次切换 7 秒延迟，之后常驻。
**结论**：**模型热切换是 LoB 的软点，但可以通过 Track A 内置模型仓来规避。**

### C5 · 多用户 tenancy

**场景**：一个 AIM 服务多台机器/多用户。
**假设风险**：边界带宽 × N 用户可能超过 USB4 容量。
**结论**：DTA 是单用户/边缘架构，多用户走云。这是范围问题，不是反例。

### C6 · 训练场景

**场景**：训练需要在多设备间同步梯度，NVLink 900 GB/s 是必要的。
**结论**：训练**明确不在 DTA 覆盖范围**。范围声明，不是反例。

### C7 · Diffusion / 图像生成大分辨率输出

**场景**：SD3 / Flux 出一张 4K 图，输出 ≈ 24 MB。
**假设风险**：USB4 5 GB/s → 5ms 传输，可以忽略。
**结论**：**不构成反例。**
